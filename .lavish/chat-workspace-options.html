Workspace redesign discussion

Make the build screen feel like a kid's studio, not a debug panel.

The screenshot has two separate problems: technical details are leaking into the kid-facing header, and the empty right side is not earning its space.

Design target

Warm arcade. Clear for parents. Friendly for ages 7-12.

What feels off

The screen is doing grown-up work in kid space.

The current layout makes implementation state more visible than the learner's goal. The workspace should answer: what am I building, what can I try next, and what is Bit doing for me?

Implementation notes

Likely code changes after we choose.

These are not final until you choose a direction, but they show where the screenshot problems come from.

Header cleanup

Update the workspace title area to use learner-facing status.

  • Remove raw `openai-codex/gpt-5.5` from the visible header.
  • Remove account id from the visible subtitle.
  • Use copy like `Ada is building a reaction game` and `Bit is ready`.

Right rail behavior

The current grid reserves a second column even when no tool panel renders.

  • Add a layout class for `tools.length === 0` if we choose full-width chat.
  • Or always render a kid-friendly rail if we choose Option B.
  • Keep raw tool output behind an explicit details disclosure.

Empty state

The first moments should offer useful next actions instead of waiting on a blank chat.

  • Add idea chips such as `Make it faster`, `Add a timer`, `Change colors`.
  • Show Bit art and a short mission card.
  • Make the composer prompt specific to the project.
