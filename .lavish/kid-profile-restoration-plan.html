Proposal

Restore kid profiles without undoing the Pi runtime work.

The last commit removed the old profile gate, profile storage service, parent shell, and profile-scoped project model. The current app signs in to Codex, then immediately lists or creates global factory projects. The plan below brings back a kid profile concept as the selected Lead Builder context for projects and Bit turns.

What changed

`src/renderer/src/App.tsx` now routes authenticated users directly to `ProjectPicker` when there is no active project.

`src/main/storage/layout.ts` now seeds a default factory and a default lead record, but the renderer cannot list, create, or select kid profiles.

What existed before

`HEAD^:src/shared/profile.ts` defined a `Profile` with `name`, `age`, `interests`, optional `notes`, `sessions`, `currentDreamId`, and `dreamHistory`.

The old renderer had `ProfileGate`, `CreateProfileForm`, `ParentShell`, and a Zustand `profileStore` backed by main-process IPC.

Core gap

Projects are currently global to the default factory.

Bit receives a generic Lead Builder prompt instead of a selected kid's name, age, interests, and parent notes.

Recommended Direction

Make kid profiles the selected Lead Builder context.

Keep Codex auth as a single parent-owned credential. After auth, require selecting or creating a kid profile before listing projects. Scope projects, chat history, logbooks, and Bit prompts to that selected profile.

Target Flow

1
Signed-out users see `AuthGate` and sign in with Codex.
2
Authenticated users load `profiles.list()` before projects.
3
No profiles shows a first-profile creation screen.
4
One or more profiles shows `ProfileGate` with kid cards and a create-profile action.
5
Selecting a profile opens that kid's project picker and chat workspace.

First MVP Non-Goals

Do not resurrect the entire old parent dashboard in the first patch unless you want that explicitly.
  • Do not bring back dream picking, mastery grids, flags, transcript audit, or parent chat yet.
  • Do not reintroduce ACPX or old harness config around profiles.
  • Do not change Codex auth storage from one local parent-owned token unless product requires per-profile auth later.
  • Do not migrate current-commit test projects or old `.hi-bit/profiles` data in this pass.
